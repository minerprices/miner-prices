🗄️ DATABASE SCHEMA (PostgreSQL)
================================================================================

TABLE: vendors
  - id (PK), email (UNIQUE), password_hash
  - company_name, contact_name, contact_phone, website
  - approved (BOOL), pre_approved (BOOL)
  - created_at, updated_at
  INDEX: approved, pre_approved

TABLE: miners
  - id (PK), whattomine_id (UNIQUE), name, algorithm
  - power_consumption, price, manufacturer, description
  - image_url, specs (JSONB)
  - is_active (BOOL), last_synced_at
  - created_at, updated_at
  INDEX: algorithm, is_active

TABLE: locations
  - id (PK), vendor_id (FK), name, city, country
  - cooling_type, power_cost_per_kwh, hosting_fee_per_kwh
  - setup_fee, bandwidth_included_mbps, available_power_kw
  - description, contact_email, contact_phone, website
  - is_active (BOOL)
  - created_at, updated_at
  INDEX: vendor_id, is_active

TABLE: approvals
  - id (PK), vendor_id (FK), admin_id
  - action (approve/reject), reason, status
  - created_at
  INDEX: vendor_id, status

TABLE: sync_log
  - id (PK), sync_type, miners_added, miners_updated, miners_total
  - status, error_message
  - created_at
  INDEX: created_at

TABLE: password_resets
  - id (PK), vendor_id (FK), token (UNIQUE)
  - expires_at, used (BOOL)
  - created_at
  INDEX: vendor_id

TABLE: admins
  - id (PK), email (UNIQUE), password_hash, name
  - role, is_active (BOOL)
  - created_at, updated_at

🚀 HOW TO GET STARTED
================================================================================

QUICK START (5 minutes)

1. Clone to your machine:
   cd /data/.openclaw/workspace/miner-prices

2. Create database:
   createdb miner_prices

3. Setup backend:
   cd backend
   npm install
   cp .env.example .env
   npm run migrate
   npm run dev

4. Setup frontend (new terminal):
   cd frontend
   npm install
   npm start

5. Test it:
   - Open http://localhost:3000
   - Browse miners/locations
   - Register as vendor
   - Admin approves
   - Add hosting location

See QUICKSTART.md for detailed instructions!
